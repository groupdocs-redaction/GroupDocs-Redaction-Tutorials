## 什么是 Secure Document Redaction？

Secure document redaction 是一种永久删除或遮蔽敏感信息（如个人标识符、财务数据或机密评论）的技术，从数字文件中移除这些信息。不同于简单的隐藏，编辑确保数据无法通过任何方式恢复，满足 GDPR、HIPAA 和 PCI‑DSS 等合规标准。

## 为什么使用 GroupDocs.Redaction？

GroupDocs.Redaction 为跨多个平台的文档编辑提供统一的 API。以下是选择我们解决方案的一些重要理由：

在 .NET 和 Java 应用中保持一致的文档编辑逻辑，降低开发时间和维护成本。

### Extensive Format Support
从 30 多种流行文档格式中编辑敏感信息，包括：

- PDF 文档
- Microsoft Office 格式（Word、Excel、PowerPoint）
- OpenDocument 格式
- 图像格式（JPEG、PNG、TIFF）
- 电子邮件格式（MSG、EML）
- 以及其他众多格式

### Comprehensive Redaction Options
- 使用精确短语、正则表达式或区分大小写的匹配来编辑文本

- 编辑可能包含机密信息的批注和评论  
- 删除包含敏感内容的整页或页范围  
- 针对特定文档类型应用自定义编辑策略  

### Privacy and Security Focused
- 永久删除敏感信息，无法恢复

- 防篡改功能，防止未授权修改  
- 完整的文档清理，包括隐藏的元数据和属性  

### No External Dependencies
GroupDocs.Redaction 在无需安装 Microsoft Office、Adobe Acrobat 或其他第三方工具的情况下即可工作。

## Frequently Asked Questions

**Q: GroupDocs.Redaction 是否支持受密码保护的文件？**  
A: 是的。您可以在加载阶段提供密码，以打开加密的 PDF、Word 或 Excel 文件。

**Q: 我可以批量编辑多个文档的内容吗？**  
A: 当然可以。API 允许您遍历文件集合，对每个文件应用相同的编辑策略。

**Q: 光栅化是可选的吗？**  
A: 是的。当您需要仅图像输出以增强安全性时，可以选择对整个文档或特定页面进行光栅化。

**Q: 支持哪些编程语言？**  
A: 核心 SDK 提供 .NET（C# / VB.NET）和 Java。其他平台也有包装库。

**Q: 如何确保编辑符合 GDPR？**  
A: 使用内置的元数据删除和永久内容删除功能；API 保证编辑后的数据无法恢复，满足 GDPR 的“被遗忘权”要求。
