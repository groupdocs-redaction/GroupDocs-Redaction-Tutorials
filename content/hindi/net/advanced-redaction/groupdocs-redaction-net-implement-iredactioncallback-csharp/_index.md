# GroupDocs.Redaction .NET (C#) के साथ संवेदनशील डेटा को रिडैक्ट करें

आज के डिजिटल परिदृश्य में, **redact sensitive data** को कानूनी, वित्तीय या HR दस्तावेज़ों से हटाना अनिवार्य आवश्यकता है। चाहे आप बाहरी समीक्षा के लिए अनुबंध तैयार कर रहे हों या सार्वजनिक रिलीज़ से पहले रिपोर्ट को साफ़ कर रहे हों, एक भी व्यक्तिगत पहचानकर्ता की कमी अनुपालन उल्लंघनों का कारण बन सकती है। GroupDocs.Redaction for .NET आपको एक शक्तिशाली, प्रोग्रामेटिक तरीका प्रदान करता है जिससे आप सुनिश्चित कर सकते हैं कि प्रत्येक गोपनीय जानकारी बिल्कुल उसी तरह गायब हो जैसा आप चाहते हैं। इस ट्यूटोरियल में हम `IRedactionCallback` इम्प्लीमेंटेशन को अटैच करने की प्रक्रिया दिखाएंगे, ताकि आप रिडैक्शन वर्कफ़्लो को कस्टमाइज़ कर सकें और हर कदम पर पूर्ण नियंत्रण रख सकें।

## त्वरित उत्तर
- **IRedactionCallback क्या करता है?** यह आपको रिडैक्शन इवेंट्स को इंटरसेप्ट करने, उन्हें लॉग करने, या ऑन‑द‑फ्लाई व्यवहार को संशोधित करने की अनुमति देता है।  
- **क्या मुझे लाइसेंस चाहिए?** विकास के लिए ट्रायल काम करता है; एक स्थायी लाइसेंस सभी इवैल्यूएशन सीमाओं को हटा देता है।  
- **कौन से .NET संस्करण समर्थित हैं?** .NET Core 3.1+, .NET 5/6, और .NET Framework 4.6+।  
- **क्या मैं कई फ़ाइलों को प्रोसेस कर सकता हूँ?** हाँ—लॉजिक को लूप में रैप करें या सर्वोत्तम प्रदर्शन के लिए बैच प्रोसेसिंग का उपयोग करें।  
- **क्या असिंक्रोनस रिडैक्शन संभव है?** बिल्ट‑इन नहीं, लेकिन आप API कॉल्स को `Task.Run` या अन्य async पैटर्न के अंदर चला सकते हैं।

## संवेदनशील डेटा को रिडैक्ट करना क्या है?
रिडैक्शन वह प्रक्रिया है जिसमें ऐसी जानकारी को स्थायी रूप से हटाया या अस्पष्ट किया जाता है जिसे प्रकट नहीं किया जाना चाहिए। GroupDocs.Redaction के साथ आप सटीक वाक्यांश, पैटर्न या कस्टम नियम निर्धारित कर सकते हैं और उन्हें प्लेसहोल्डर (जैसे **[REDACTED]**) से बदल सकते हैं जबकि मूल दस्तावेज़ लेआउट बरकरार रहता है।

## IRedactionCallback के साथ GroupDocs.Redaction क्यों उपयोग करें?
- **पूर्ण ऑडिटेबिलिटी:** कॉलबैक आपको किए गए प्रत्येक रिडैक्शन का विस्तृत लॉग देता है।  
- **कस्टम हैंडलिंग:** टेक्स्ट बदलें, बाहरी सेवाओं को ट्रिगर करें, या डायनामिक रूप से बिज़नेस नियम लागू करें।  
- **स्केलेबल परफॉर्मेंस:** बैच प्रोसेसिंग के साथ कॉलबैक्स को मिलाकर हजारों फ़ाइलों को कुशलता से संभालें।

## इम्प्लीमेंटेशन गाइड
अब हम बेसिक सेटअप को एक कस्टम `IRedactionCallback` जोड़कर विस्तारित करेंगे। इससे आप प्रत्येक रिडैक्शन इवेंट को कैप्चर कर सकते हैं, लॉग में लिख सकते हैं, या रीयप्लेसमेंट टेक्स्ट को ऑन‑द‑फ्लाई बदल सकते हैं।

### IRedactionCallback इम्प्लीमेंटेशन को अटैच और उपयोग करें
निम्न चरण पूर्ण वर्कफ़्लो दिखाते हैं, फ़ाइल पाथ तैयार करने से लेकर सटीक‑फ़्रेज़ रिडैक्शन लागू करने तक।

#### चरण 1: आउटपुट डायरेक्टरी और स्रोत फ़ाइल पाथ तैयार करें
परिभाषित करें कि आपका स्रोत दस्तावेज़ कहाँ स्थित है। अपने वातावरण के अनुसार पाथ को समायोजित करें।

```csharp
string sourceFile = "YOUR_DOCUMENT_DIRECTORY/sample.docx"; // Replace with actual path
```

#### चरण 2: कस्टम सेटिंग्स के साथ Redactor इंस्टेंस बनाएं
हम `Redactor` को `LoadOptions` और `RedactorSettings` के साथ इंस्टैंशिएट करते हैं। सेटिंग्स के भीतर `RedactionDump` स्वचालित रूप से प्रत्येक रिडैक्शन को रिकॉर्ड करेगा।

```csharp
using (Redactor redactor = new Redactor(sourceFile, 
    new LoadOptions(), 
    new RedactorSettings(new RedactionDump())))
{
    // Further processing steps will go here
}
```

#### चरण 3: एक सटीक वाक्यांश रिडैक्शन लागू करें
यहाँ हम **John Doe** वाक्यांश को प्लेसहोल्डर **[REDACTED]** से बदलते हैं। आप अपनी आवश्यकता के अनुसार कोई भी वाक्यांश या पैटर्न स्वैप कर सकते हैं।

```csharp
redactor.Apply(new ExactPhraseRedaction("John Doe", new ReplacementOptions("[REDACTED]")));
```

**मुख्य ऑब्जेक्ट्स की व्याख्या:**
- `LoadOptions()` – SDK को बताता है कि दस्तावेज़ को कैसे पढ़ा जाए (जैसे पासवर्ड हैंडलिंग)।  
- `RedactorSettings(new RedactionDump())` – एक डम्प फ़ाइल सक्षम करता है जो ऑडिट उद्देश्यों के लिए प्रत्येक रिडैक्शन को लॉग करती है।  
- `ReplacementOptions("[REDACTED]")` – वह टेक्स्ट परिभाषित करता है जो मेल खाने वाले वाक्यांश को बदल देगा।

### यह क्यों महत्वपूर्ण है
`IRedactionCallback` का उपयोग करके आप कस्टम लॉजिक प्लग इन कर सकते हैं, जैसे:
- रिडैक्शन विवरण को एक अनुपालन डेटाबेस में भेजना।  
- साधारण वाक्यांश प्रतिस्थापन में न शामिल अतिरिक्त मेटाडाटा को मास्क करना।  
- कंटेंट टाइप के आधार पर डायनामिक रूप से प्रतिस्थापन टेक्स्ट चुनना।

### समस्या निवारण टिप्स
- **File not found:** `sourceFile` पाथ को दोबारा जांचें और सुनिश्चित करें कि फ़ाइल चलाने वाली प्रक्रिया के लिए सुलभ है।  
- **Callback not firing:** पुष्टि करें कि आपका क्लास `IRedactionCallback` के **सभी** सदस्य को इम्प्लीमेंट करता है और इंस्टेंस सही ढंग से `Redactor` को पास किया गया है।  
- **Performance lag:** बड़े बैच के लिए संभव हो तो वही `Redactor` इंस्टेंस पुनः उपयोग करें और तुरंत डिस्पोज़ करें।

## व्यावहारिक अनुप्रयोग
संवेदनशील डेटा को रिडैक्ट करना कई उद्योगों में उपयोगी है:

1. **Legal Document Processing** – ड्राफ्ट साझा करने से पहले क्लाइंट नाम, केस नंबर या सोशल सिक्योरिटी नंबर स्वचालित रूप से हटाएँ।  
2. **HR Management Systems** – ऑडिट के दौरान कर्मचारी अनुबंधों से व्यक्तिगत पहचानकर्ता हटाएँ।  
3. **Financial Reporting** – निवेशक‑फ़ेसिंग PDFs बनाते समय स्वामित्व वाले आंकड़े या अकाउंट नंबर छिपाएँ।

## प्रदर्शन विचार
दसियों या सैकड़ों फ़ाइलों को संभालते समय अपने एप्लिकेशन को तेज़ रखने के लिए:

- **Batch Processing:** फ़ाइलों की सूची लोड करें और `Parallel.ForEach` के अंदर रिडैक्शन लूप चलाएँ ताकि मल्टी‑कोर उपयोग हो सके।  
- **Memory Management:** प्रत्येक `Redactor` को `using` ब्लॉक में रैप करें (जैसा दिखाया गया) ताकि डिस्पोज़ सुनिश्चित हो।  
- **Asynchronous Operations:** जबकि SDK स्वयं सिंक्रोनस है, आप कार्य को बैकग्राउंड थ्रेड्स या `Task.Run` में ऑफ़लोड कर सकते हैं ताकि UI थ्रेड ब्लॉक न हो।

## सामान्य समस्याएँ और समाधान
| Issue | Solution |
|-------|----------|
| **“Invalid file format” error** | सुनिश्चित करें कि दस्तावेज़ प्रकार समर्थित है (PDF, DOCX, PPTX, आदि)। |
| **Callback receives null values** | जांचें कि `RedactorSettings` बनाते समय आप `IRedactionCallback` की एक ठोस इम्प्लीमेंटेशन पास कर रहे हैं। |
| **Redaction not applied** | सत्यापित करें कि सटीक वाक्यांश दस्तावेज़ के केस और स्पेसिंग से मेल खाता है, या पैटर्न‑आधारित मिलान के लिए `RegexRedaction` का उपयोग करें। |

## अक्सर पूछे जाने वाले प्रश्न

**Q: GroupDocs.Redaction के लाइसेंस विकल्प क्या हैं?**  
A: आप फ्री ट्रायल से शुरू कर सकते हैं या सभी फीचर्स का अन्वेषण करने के लिए टेम्पररी लाइसेंस अनुरोध कर सकते हैं। प्रोडक्शन के लिए, स्थायी या सब्सक्रिप्शन लाइसेंस खरीदें।

**Q: क्या मैं GroupDocs.Redaction को कई फ़ाइल प्रकारों पर उपयोग कर सकता हूँ?**  
A: हाँ, यह PDFs, Word, Excel, PowerPoint और कई अन्य सामान्य फ़ॉर्मेट्स को सपोर्ट करता है।

**Q: रिडैक्शन के दौरान अपवादों को कैसे हैंडल करूँ?**  
A: अपने रिडैक्शन लॉजिक को `try‑catch` ब्लॉक्स में रैप करें और अपवाद विवरण लॉग करें। कॉलबैक को रियल‑टाइम में त्रुटियों को कैप्चर करने के लिए भी उपयोग किया जा सकता है।

**Q: क्या असिंक्रोनस प्रोसेसिंग के लिए बिल्ट‑इन सपोर्ट है?**  
A: कोर API सिंक्रोनस है, लेकिन आप रिडैक्शन कॉल्स को असिंक्रोनस टास्क या बैकग्राउंड सर्विसेज में चला सकते हैं।

**Q: अधिक उन्नत उदाहरण कहाँ मिल सकते हैं?**  
A: आधिकारिक [documentation](https://docs.groupdocs.com/redaction/net/) और API रेफ़रेंस में विस्तृत कोड सैंपल और परिदृश्य गाइड उपलब्ध हैं।
