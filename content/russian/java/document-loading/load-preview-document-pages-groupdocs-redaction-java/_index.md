### Загрузка и предварительный просмотр страницы документа

#### Обзор
В этом разделе показано, как создать PNG‑превью определённой страницы документа с помощью GroupDocs.Redaction для Java. Это особенно полезно для быстрых проверок или создания миниатюрных изображений документов.

##### Шаг 1: Установите номер целевой страницы
Начните с указания, какую страницу вы хотите просмотреть:

```java
int testPageNumber = 1;
```

Это задаёт `testPageNumber` равным 1, что означает генерацию превью первой страницы.

##### Шаг 2: Определите путь к выходному файлу
Укажите, где должен быть сохранён сгенерированный PNG‑файл. Используйте плейсхолдеры для динамических имён файлов:

```java
final String previewFileName = "YOUR_OUTPUT_DIRECTORY_page%d.png";
```

Такой формат позволяет динамически задавать имя файла в зависимости от номера страницы.

##### Шаг 3: Настройте параметры предварительного просмотра
Настройте `PreviewOptions`, чтобы определить, как будет создаваться и сохраняться превью. Реализуйте интерфейс `ICreatePageStream` для пользовательского создания потоков:

```java
PreviewOptions options = new PreviewOptions(new ICreatePageStream() {
    @Override
    public java.io.OutputStream createPageStream(int pageNumber) {
        try {
            return new java.io.FileOutputStream(java.lang.String.format(previewFileName, pageNumber));
        } catch (Exception e) {
            // Handle exceptions related to file stream creation
            e.printStackTrace();
            return null;
        }
    }
});
options.setPreviewFormat(PreviewFormats.PNG);
options.setPageNumbers(new int[] { testPageNumber });
```

- **ICreatePageStream**: этот интерфейс позволяет создавать пользовательский выходной поток для каждой страницы.  
- **setPreviewFormat**: укажите формат превью; в данном случае PNG.  
- **setPageNumbers**: задайте, какие страницы должны быть сгенерированы как превью.

#### Советы по устранению неполадок
- Убедитесь, что пути к файлам указаны корректно и доступны вашему приложению.  
- Обрабатывайте исключения должным образом, чтобы избежать ошибок выполнения во время создания потоков.
